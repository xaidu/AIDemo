# Code Review Checklist

A comprehensive checklist for conducting thorough code reviews.

## Functionality
- [ ] Code does what it's supposed to do
- [ ] Logic is correct and handles edge cases
- [ ] Error handling is appropriate
- [ ] No obvious bugs or logical errors
- [ ] Performance considerations are addressed

## Code Quality
- [ ] Code is readable and well-structured
- [ ] Variable and function names are descriptive
- [ ] Functions are reasonably sized and focused
- [ ] Code follows established patterns and conventions
- [ ] No unnecessary complexity or over-engineering

## Security
- [ ] No sensitive information exposed
- [ ] Input validation is proper
- [ ] SQL injection and XSS vulnerabilities addressed
- [ ] Authentication and authorization checked
- [ ] Secure coding practices followed

## Testing
- [ ] Tests are included for new functionality
- [ ] Existing tests still pass
- [ ] Test coverage is adequate
- [ ] Tests are meaningful and test the right things
- [ ] Edge cases are covered

## Documentation
- [ ] Code is properly commented where necessary
- [ ] Documentation is updated if needed
- [ ] API changes are documented
- [ ] Breaking changes are clearly noted
- [ ] README or other docs updated as needed

## Dependencies
- [ ] New dependencies are justified and secure
- [ ] Dependency versions are specified
- [ ] No unnecessary dependencies added
- [ ] License compatibility checked
